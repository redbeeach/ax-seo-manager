프로젝트 소개

AX SEO Manager는 단순한 SEO 검사기를 넘어, AI 검색 시대에 맞는 콘텐츠 품질 진단 + 개선 가이드 플랫폼입니다.

🔍 DB 기반 분석
저장된 메타데이터 기준으로 즉시 점수 산출. 1초 이내 확인.
🌐 Live 크롤링 분석
실제 배포된 페이지 HTML을 직접 가져와서 검색엔진 관점으로 정밀 분석.
🤖 AI 자동 최적화
GPT 기반으로 SEO Title, Meta, FAQ, GEO 요약, JSON-LD 자동 생성.
💡 그누보드5(GB5)와 연동되어 게시글 작성/수정 시 자동 동기화됩니다. write_update.php 훅으로 콘텐츠가 자동으로 등록됩니다.
2
기본 워크플로우
1
콘텐츠 등록
GB5 게시판에 글 작성 시 자동 동기화. 또는 콘텐츠 목록 → + 새 글로 직접 등록.
2
AI 최적화 실행
콘텐츠 상세 → AI 최적화 실행 버튼 클릭. SEO Title / Meta / FAQ / GEO 요약 / JSON-LD 자동 생성.
3
실제 페이지 크롤링
🔄 실제 페이지 크롤링 버튼으로 배포된 페이지 기준 정밀 분석. 결과는 DB에 저장되어 새로고침 후에도 유지.
4
개선 추천 확인
🔥 개선 추천 섹션에서 영향도 순으로 정렬된 항목 확인. 예상 점수 상승 (+N pt) 확인 후 우선순위대로 수정.
5
Entity / Semantic 분석
AI 분석 실행 버튼으로 핵심 개체 추출 + 주제 관련 키워드 포함률 확인. 결과 DB 저장.
3
Overall Score + Grade

7개 점수(SEO·AEO·GEO·Content·AI Citation·E-E-A-T·Readability)의 평균값입니다.

Grade점수 범위의미
A+90 ~ 100최우수 — AI 검색 최적화가 잘 되어 있습니다.
A80 ~ 89우수 — 일부 항목만 보완하면 최상위권입니다.
B70 ~ 79양호 — 핵심 개선 항목을 수정하면 크게 올라갑니다.
C60 ~ 69보통 — AI 검색 노출을 위해 개선이 필요합니다.
D50 ~ 59미흡 — 여러 핵심 항목이 누락되어 있습니다.
F0 ~ 49불량 — 기본 SEO 요소부터 점검이 필요합니다.

SEO Title 60자 이하15pt60자 초과 시 Google 검색 결과에서 잘림
Meta Description 존재15ptmeta_description 필드가 채워진 경우
Meta Description 155자 이하15pt155자 초과 시 잘림
OG 태그 존재15ptog_title + og_description 모두 있는 경우
Canonical URL 설정15ptcanonical_url, page_slug, GB5 게시판 정보 중 하나라도 있는 경우
robots index/follow 허용10ptrobots_index, robots_follow가 false가 아닌 경우
5
AEO 점수 (100점)

Answer Engine Optimization. ChatGPT, Perplexity 같은 AI가 답변에 인용하기 좋은 구조인지 평가합니다.

항목배점기준
FAQ 3개 이상40ptfaq_json에 3개 이상 질문-답변이 있는 경우
한 줄 답변 존재30ptae_answer 필드가 채워진 경우
Q&A 구조 존재30ptFAQ가 1개 이상 있는 경우
💡 AI 최적화 실행 시 FAQ와 한 줄 답변이 자동 생성됩니다.
6
GEO 점수 (100점)

Generative Engine Optimization. AI가 생성하는 답변에 인용될 가능성을 평가합니다.

항목배점기준
요약 존재30ptgeo_summary 필드가 채워진 경우
JSON-LD 존재40ptjson_ld 필드에 구조화 데이터가 있는 경우
전문성(요약 50자 이상)30ptgeo_summary가 50자 이상인 경우
7
Content 점수 (100점)

본문 HTML 구조를 직접 파싱해서 검사합니다. 🟢 Live 크롤링 시 실제 배포 페이지 기준으로 갱신됩니다.

항목배점기준
H1 존재15pt페이지 제목이 H1으로 마크업된 경우. 제목(title)이 있으면 자동 통과
H1 1개만 사용10ptH1이 2개 이상이면 중복으로 감점
H2 2개 이상15pt본문 섹션 구분을 H2로 나눈 경우
이미지 ALT 누락 0개20pt모든 이미지에 ALT 텍스트가 있는 경우. 이미지 없으면 자동 통과
내부링크 3개 이상15pt같은 도메인 링크가 3개 이상인 경우
본문 1000자 이상15ptHTML 태그 제거 후 순수 텍스트 기준
표 또는 목차 존재10pt<table> 태그 또는 "목차" 텍스트가 있는 경우
8
AI Citation 점수 (100점)

ChatGPT가 이 글을 답변에 인용하기 쉬운 구조인지 평가합니다.

항목배점기준
FAQ 존재15ptQ&A 형태 콘텐츠가 있는 경우
요약 존재15ptGEO 요약이 채워진 경우
표 존재10pt<table> 태그가 있는 경우
리스트 존재10pt<ul> 또는 <ol> 태그가 있는 경우
신뢰 가능한 출처(외부링크) 존재15pt외부 도메인 링크가 있는 경우
정의문 존재(도입부)10pt"~란", "~이란", "~는 무엇" 패턴이 첫 400자 안에 있는 경우
H2 섹션 구조(2개 이상)10ptH2가 2개 이상인 경우
출처 표기15pt<blockquote>, <cite> 태그 또는 "출처:", "참고:" 텍스트가 있는 경우
9
E-E-A-T 점수 (100점)

Google이 중요하게 보는 경험(Experience)·전문성(Expertise)·권위성(Authoritativeness)·신뢰성(Trustworthiness) 지표입니다.

항목배점기준
작성자(author) 명시20ptJSON-LD에 author.name이 있는 경우
발행일(datePublished) 명시15ptJSON-LD에 datePublished가 있는 경우
수정일(dateModified) 명시15ptJSON-LD에 dateModified가 있는 경우
발행처(publisher) 명시15ptJSON-LD에 publisher.name이 있는 경우
본문 깊이(2000자 이상)20pt순수 텍스트 기준 2000자 이상인 경우
외부 출처 링크 존재15pt외부 도메인 링크가 있는 경우
💡 AI 최적화 실행 시 JSON-LD에 author, datePublished, dateModified, publisher가 자동으로 포함됩니다.
10
Readability 점수 (100점)

문장 가독성을 측정합니다. 짧고 명확한 문장이 독자와 AI 모두 읽기 좋습니다.

항목배점기준
평균 문장 길이 (15~50자)30pt너무 짧거나 길면 부분 점수(10pt)
긴 문장(80자↑) 비율 20% 미만25pt80자 이상 문장이 전체의 20% 미만인 경우
문장 수 5개 이상20pt분석 의미가 있는 충분한 분량
단락 수 3개 이상25pt<p> 또는 <br><br> 기준으로 단락 구분
11
실제 페이지 크롤링

배포된 실제 HTML을 직접 가져와서 분석합니다. DB에 저장되어 새로고침 후에도 유지됩니다.

DB 기준 vs Live 기준
DB 기준 Supabase에 저장된 title, body 기반으로 즉시 계산. 1초 이내.
🟢 Live 기준 실제 배포 URL을 서버에서 fetch → HTML 파싱 → Content 점수 갱신. 결과 DB 저장.
  • 크롤링 완료 시 Content 점수와 Overall이 실제 기준으로 갱신됩니다
  • 결과는 content_live_analyses 테이블에 저장 (새로고침 유지)
  • 콘텐츠 목록에서 크롤링된 글은 🟢 Live 뱃지로 표시됩니다
  • JS 동적 렌더링 사이트(React SPA 등)는 HTML이 비어있어 정확도가 낮을 수 있습니다
12
AI 최적화

GPT를 사용해 SEO/AEO/GEO 항목을 자동으로 생성합니다.

생성 항목내용
SEO Title키워드를 앞에 배치한 60자 이내 검색 제목
Meta Description클릭을 유도하는 155자 이내 설명문
OG Title / OG DescriptionSNS 공유 시 표시되는 제목과 설명
FAQ3개 이상 질문-답변 (AEO 핵심)
AEO 한 줄 답변AI가 바로 인용할 수 있는 핵심 요약 1문장
GEO 요약AI 생성 답변에 인용되기 좋은 전문적 요약
JSON-LDauthor, datePublished, dateModified, publisher, FAQ 포함 구조화 데이터
💡 현재 gpt-3.5-turbo 사용 중. OpenAI 조직 인증 완료 후 gpt-4o-mini로 업그레이드 예정.
13
Entity / Semantic 분석

룰베이스(H태그/TF) 사전 추출 + AI 후처리로 Entity를 분류하고 Semantic 커버리지를 측정합니다.

🧩 Entity 추출
본문에 실제 등장하는 핵심 개체를 유형별로 분류합니다.

👤 인물 · 🏢 조직 · 📍 장소 · 💻 기술 · 📦 제품 · 📅 이벤트
📊 Semantic 커버리지
이 주제의 글이라면 있어야 할 관련 단어가 실제로 포함됐는지 측정합니다. ✓ 포함 / ✗ 누락으로 표시.
  • 분석 결과는 content_entity_analyses 테이블에 저장 (새로고침 유지)
  • 크롤링 후 분석하면 실제 배포 페이지 기준으로 더 정확한 결과를 얻을 수 있습니다
  • 누락된 Semantic 키워드 = 본문에 추가하면 좋은 키워드 목록으로 활용하세요
14
경쟁 페이지 비교

경쟁사 URL을 직접 입력하면 실시간 크롤링 후 내 페이지와 점수를 비교합니다. (최대 3개)

비교 항목설명
Content 점수H태그 구조, 이미지 ALT, 내부링크, 글자수 등
AI Citation 점수FAQ, 표, 리스트, 출처 등 AI 인용 요소
Readability 점수평균 문장 길이, 긴 문장 비율, 단락 수
Gap내 페이지 대비 평균 점수 차이 (+/- 표시)
💡 나무위키, 위키백과 등 SSR 기반 사이트는 정확하게 분석됩니다. React SPA 등 CSR 사이트는 결과가 부정확할 수 있습니다.
15
점수 히스토리

콘텐츠 상세 페이지에 방문할 때마다 하루 1회 점수가 자동 저장됩니다.

  • 7개 점수(SEO·AEO·GEO·Content·Citation·E-E-A-T·Readability) + Overall 기록
  • 2일 이상 데이터 쌓이면 Recharts 라인 차트로 시각화
  • 카테고리별 선 On/Off 가능
  • 첫 기록 대비 Overall 변화량 (+N점) 표시
  • 저장 테이블: content_score_history (하루 1회 unique index)
